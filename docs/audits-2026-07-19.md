# Trois audits — sécurité, interface, cohérence du code (2026-07-19)

> Menés en parallèle sur `main` @ `8df0a42`, après les chantiers CM6, chat,
> slides et livre. Lecture seule ; l'audit d'interface a réellement piloté
> l'application (30 captures, trois types de projets, deux thèmes, trois
> langues).

## Verdict d'ensemble

Les trois audits convergent sur le même diagnostic : **les fondations sont
saines, les défauts sont des contournements de garde-fous existants.**

- **Sécurité** : frontière Electron correctement verrouillée, aucune XSS
  (tous les sinks HTML assainis, y compris le cas piégeux du surlignage
  d'entités appliqué après DOMPurify), `spawn` sans shell partout sauf une
  ligne, CSP stricte. Les failles relevées sont des ruptures de défense en
  profondeur, pas des portes ouvertes.
- **Code** : l'architecture voulue est tenue — `src/editor/` rigoureusement
  pur, plus de types dupliqués, aucun handler IPC orphelin, conversions
  regex → arbre Lezer complètes. La dette est dans les zones *non touchées*
  par les chantiers récents.
- **Interface** : les nouveautés (navigateur de chapitres, frontières de
  slides, confirmations destructives) sont réussies ; les défauts sont
  concentrés sur le premier contact et l'accessibilité.

## Un recoupement instructif

Le test de parité i18n passe **alors que les deux modales d'export sont
entièrement en français codé en dur**. Le test compare les fichiers de
locales entre eux : il ne peut pas voir qu'un composant n'appelle jamais
`t()`. Une vérification verte a donc masqué le défaut le plus visible pour
un utilisateur non francophone — au même titre que les gardes SQLite
masquaient 8 échecs réels. **Leçon commune : nos filets vérifient ce qu'ils
savent voir.**

## Ce que les audits confirment de bon

  métadonnées échappées avant LaTeX.
- `src/editor/` pur, y compris dans les modules de cette semaine.
- Une seule erreur avalée sans trace dans tout le dépôt.
- Thème clair correct (contrastes vérifiés) ; fr et de complètes hors
  modales d'export ; lien d'évitement présent et fonctionnel.
- La confirmation de retrait d'un chapitre (« le fichier reste sur le
  disque et peut être rattaché ») est citée comme modèle à imiter.
- Le renderer est passé de ~79 `any` à 8.
